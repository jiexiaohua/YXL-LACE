## YXL 打包发布交接文档（macOS DMG / Windows EXE / Linux 可选）

> 目标：让同事的 AI 在不同平台上**可复现**地打包出可分发产物，并完成最小自测。
>
> 本文默认在项目根目录执行（能看到 `src/`、`requirements.txt`、`run_ui.py`、`assets/`）。

---

## 0) 重要说明（请先读）

- **GUI 入口**：`run_ui.py`（PyInstaller 入口），内部调用 `yxl_lace.ui_qt.app:main`。
- **网络模型**：程序使用 **UDP 单端口复用**（`SessionManager`），支持**对端关闭同步退出（close/ack）**。
- **GUI 功能**：Qt GUI 支持**中英切换**（写入 `~/.yxl_lace/lang`）。
- **联系人**：联系人 **ID 自动生成并隐藏**，支持**修改 label**。

## 1) 代码获取

任选一种方式：

### A. git

```bash
git clone <仓库地址>
cd YXL-LACE
git checkout <分支名或提交号>
```

### B. zip

下载 zip 并解压，进入项目根目录。

---

## 2) 创建并激活 venv（推荐）

macOS / Linux：

```bash
python3 --version
python3 -m venv .venv
source .venv/bin/activate
```

Windows（PowerShell）：

```powershell
py -3 --version
py -3 -m venv .venv
.\.venv\Scripts\Activate.ps1
```

---

## 3)（强烈建议）清理代理环境变量再装依赖

如果 pip 报错类似 “Missing dependencies for SOCKS support”，先执行：

macOS / Linux（bash/zsh）：

```bash
for _v in ALL_PROXY all_proxy HTTP_PROXY HTTPS_PROXY http_proxy https_proxy; do
  unset "${_v}" 2>/dev/null || true
done
```

Windows（PowerShell）：

```powershell
$env:ALL_PROXY=""; $env:HTTP_PROXY=""; $env:HTTPS_PROXY=""
```

---

## 4) 安装依赖（必须）

```bash
python -m pip install -r requirements.txt -r requirements-gui.txt pyinstaller
```

说明：
- `requirements.txt`：核心加密依赖
- `requirements-gui.txt`：GUI 依赖（PySide6）
- `pyinstaller`：打包工具

---

## 5) 打包前最小自测（强烈建议）

### 5.1 启动 Qt GUI

```bash
PYTHONPATH=src python -m yxl_lace.ui_qt.app
```

检查点：
- 能正常打开窗口
- 语言下拉切换中/英，UI 文案会刷新

### 5.2 两端互联最小测试（功能回归）

两台机器（或同机两终端）各自生成密钥后建立连接，测试：
- 发送消息正常
- 一端点击 Disconnect / 关闭窗口 / 或 CLI `/quit` 后，另一端应**自动关闭会话（tab 消失/提示对端关闭）**

## 6) macOS 打包：生成 `.app`

> 注意：不同 CPU 架构必须在对应机器上打包（arm64 在 Apple Silicon；x86_64 在 Intel）。

### 6.1 架构确认

```bash
uname -m
```

输出：
- Apple Silicon：`arm64`
- Intel：`x86_64`

### 6.2 用 PyInstaller 生成 `.app`

在项目根目录：

```bash
PYTHONPATH=src python -m PyInstaller --noconfirm \
  --windowed \
  --name "YXL" \
  --paths src \
  --icon assets/app.icns \
  --add-data "assets/app.png:assets" \
  run_ui.py
```

产物：
- `dist/YXL.app`

---

## 7) macOS 打包：生成 `.dmg`

建议按架构命名 DMG（可选）：
- `dist/YXL-arm64.dmg`
- `dist/YXL-x86_64.dmg`

若只需要一个统一文件名（推荐给同事/交付）：`dist/YXL.dmg`

命令（以统一文件名 `dist/YXL.dmg` 为例）：

```bash
rm -rf dist/dmg_stage
mkdir -p dist/dmg_stage
cp -R dist/YXL.app dist/dmg_stage/
ln -s /Applications dist/dmg_stage/Applications

hdiutil create -volname "YXL" \
  -srcfolder dist/dmg_stage \
  -ov -format UDZO \
  dist/YXL.dmg
```

## 8) Windows 打包：生成 `.exe`（onefile）

> 在 Windows 机器上执行（不要跨平台打包）。

### 8.1 安装依赖同第 2～4 节

### 8.2 打包命令

在项目根目录（PowerShell）：

```powershell
$env:PYTHONPATH="src"
python -m PyInstaller --noconfirm --onefile --windowed `
  --name "YXL" `
  --paths src `
  --icon assets/app.ico `
  --add-data "assets/app.png;assets" `
  run_ui.py
```

产物：
- `dist\YXL.exe`

说明：
- Windows 图标最好准备 `assets/app.ico`（如果没有，可先去掉 `--icon`）。
- `--add-data` 的分隔符 Windows 用 `;`（不是 `:`）。

## 9) Linux（可选）

最小可用产物：PyInstaller onefile 或 onedir：

```bash
PYTHONPATH=src python -m PyInstaller --noconfirm --onefile --name "YXL" run_ui.py
```

## 10) 常见问题

### 10.1 pip 报 SOCKS 相关错误
按第 3 节清理代理环境变量后重试。

### 10.2 macOS 提示“无法打开/来自身份不明开发者”
未签名 DMG 属正常现象：
- 系统设置 → 隐私与安全 → 仍要打开
或：

```bash
xattr -dr com.apple.quarantine "/Applications/YXL-LACE-GUI.app"
```
